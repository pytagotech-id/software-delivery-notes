# Software Delivery Notes

Practical notes and templates for scoping business software before a team starts building screens.

This repository is maintained by [Pytagotech](https://www.pytagotech.com/en), a software development team based in Indonesia. We work on internal tools, dashboards, portals, inventory systems, approval workflows, and mobile-backed business software.

The goal of this repo is simple: make early software scope easier to inspect.

## Why this exists

Many custom software projects start with a feature list:

- login
- dashboard
- user roles
- reports
- notifications
- export
- mobile app

Those features may be useful, but they do not explain the workflow yet.

Before building, it helps to document:

- who starts the process
- what data is captured
- which status changes
- who reviews or approves
- what output the business needs
- which part can wait until version two
